# Configuration

pve-compose uses two configuration files: a per-project `lxc.json` and a global `pve-compose.json`.

## `lxc.json` - Per-project config

Located in each project directory alongside `docker-compose.yml`. Auto-generated by `pve-compose plan` or `pve-compose up` (if missing).

### Full schema

```json
{
  "hostname": "immich",
  "ctid": 210,
  "template": 9000,
  "storage": "local-zfs",
  "disk": "8G",
  "cores": 1,
  "memory": 1024,
  "swap": 512,
  "ipv4": "dhcp",
  "gateway": "192.168.1.1",
  "dns": "1.1.1.1",
  "bridge": "vmbr0",
  "privileged": true,
  "tags": ["{ipv4}", "pve-compose"],
  "features": {
    "nesting": true,
    "keyctl": false
  },
  "mount": {
    "source": "/data/app/immich",
    "target": "/data"
  }
}
```

### Field reference

| Field | Type | Default | Description |
|---|---|---|---|
| `hostname` | string | directory name | LXC hostname. Derived from `basename $PWD` |
| `ctid` | integer | next available | Container ID. Auto-detected via Proxmox API |
| `template` | int/string | auto | Template CTID (e.g., `9000`) or OS tarball filename |
| `storage` | string | auto | Storage pool for rootfs. Auto-detected (zfspool > lvmthin > dir) |
| `disk` | string | `"8G"` | Root disk size. Data lives on bind mount, so 8G is usually enough |
| `cores` | integer | `1` | CPU cores |
| `memory` | integer | `1024` | RAM in MB |
| `swap` | integer | `512` | Swap in MB |
| `ipv4` | string | `"dhcp"` | IPv4 address. Use `"dhcp"` or `"192.168.1.100/24"` |
| `gateway` | string | auto | Default gateway. Auto-detected from host |
| `dns` | string | auto | DNS server. Auto-detected from host (skips localhost) |
| `bridge` | string | `"vmbr0"` | Network bridge |
| `privileged` | boolean | `true` | Privileged container. Simplifies Docker setup |
| `tags` | array | `["{ipv4}", "pve-compose"]` | Proxmox tags. Supports `{var}` templates |
| `features` | object | derived | LXC features. Derived from `privileged` field |
| `mount.source` | string | `$PWD` | Host directory to bind-mount |
| `mount.target` | string | `"/data"` | Mount point inside the container |

### Resolution chain

Every field is resolved through a priority chain:

```
lxc.json value -> global config default -> auto-detection -> hardcoded fallback
```

For example, `storage`:
1. If `lxc.json` has `"storage": "local-zfs"` -> use that
2. Else if `pve-compose.json` has `"defaults.storage": "local-zfs"` -> use that
3. Else auto-detect from host (priority: zfspool > lvmthin > lvm > dir)
4. Else fallback to `"local"`

### Tag templates

Tags support `{var}` placeholders that expand at deploy time:

| Template | Expands to |
|---|---|
| `{hostname}` | Container hostname |
| `{ctid}` | Container ID |
| `{ipv4}` | IPv4 address (filtered: `dhcp` is excluded) |

Tags with unresolved `{var}` or invalid characters are automatically filtered.

### Features derivation

The `features` field is derived from `privileged`:

- `privileged: true` -> `nesting=1` (Docker needs namespace nesting)
- `privileged: false` -> `nesting=1,keyctl=1` (unprivileged also needs keyctl)

You generally don't need to set `features` manually.

## `pve-compose.json` - Global config

Located at `/etc/pve-compose/pve-compose.json`. Created by `pve-compose setup`.

### Full schema

```json
{
  "template": {
    "ctid": 9000
  },
  "defaults": {
    "storage": "local-zfs",
    "disk": "8G",
    "cores": 1,
    "memory": 1024,
    "swap": 512,
    "bridge": "vmbr0",
    "ipv4": "dhcp",
    "dns": "1.1.1.1",
    "gateway": "192.168.1.1"
  },
  "mount": {
    "target": "/data"
  },
  "ctid": {
    "range": [100, 8999],
    "template_range": [9000, 9099]
  }
}
```

### Sections

**`template`**: Template settings. `ctid` is set automatically by `pve-compose template create`.

**`defaults`**: Default values used when `lxc.json` doesn't specify a field. All values in this block can be overridden per-project.

**`mount`**: Default mount target inside containers.

**`ctid`**: ID ranges. `range` for regular containers, `template_range` for templates.

## Project directory structure

```
/data/app/speedtest/
├── docker-compose.yml    # your compose file (required)
├── lxc.json              # auto-generated config (editable)
└── .pve-compose/         # internal state
    └── bootstrap.done    # Docker install marker
```

The only file you **must** provide is `docker-compose.yml`. Everything else is auto-generated.

## Environment detection

`pve-compose setup` and the resolution engine detect these values from your host:

| Value | Source | Method |
|---|---|---|
| Storage | `/etc/pve/storage.cfg` | Parse config, priority: zfspool > lvmthin > dir |
| Gateway | `ip route` | Default route gateway |
| DNS | `/etc/resolv.conf` | First non-localhost nameserver |
| Bridge | `ip link` | First bridge interface, fallback `vmbr0` |
| Template | Global config | `template.ctid` field set by `template create` |
| Next CTID | Proxmox API | `pvesh get /cluster/nextid` |

DNS detection specifically filters out `127.x.x.x` and `::1` addresses (common with Pi-hole/AdGuard setups), since those don't work inside LXC containers.
